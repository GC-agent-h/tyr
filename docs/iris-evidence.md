# Phase 00 — Evidence: Iris replication backend is in use

**Conclusion: CONFIRMED** (static/file-based evidence; no live debugger was
used — none is available on this project, see `open-assumptions.md` A2).

This is a load-bearing fact. It drives the Iris-specific rewrites of Phases
04, 06, and 07 (`FNetRefHandle`/`FNetToken` resolution, protocol/descriptor
schema cache, Iris `NetSerializer`s, Iris `FNetBlob` RPCs) rather than the
legacy `FRepLayout`/`NetFieldExportGroup`/`FNetworkGUID` path.

## Evidence gathered (all from files already present in this repo)

### E1 — IrisCore package is compiled into the game SDK
`dumper-7/CppSDK/SDK/IrisCore_classes.hpp` exists and is generated by
Dumper-7 from the running build. Its header declares `// Package: IrisCore`
and concrete Iris types such as `UNetObjectFilterConfig`,
`UAlwaysRelevantNetObjectFilterConfig`, etc. A compiled-in IrisCore package
proves the engine build was linked with Iris.

### E2 — Iris runtime objects are instantiated in the live game
`dumper-7/GObjects-Dump.txt` (a dump of the running game's `GUObjectArray`)
contains the following Iris subsystem identifiers:

| Token | Occurrences |
|---|---|
| `IrisCore` | 184 |
| `IrisFastArraySerializer` | 1 |
| `IrisNetDriverConfig` | 1 |
| `IrisObjectReferencePackageMap` | 1 |
| `ObjectReplicationBridge` | 11 |
| `ReplicationBridge` | 6 |
| `ReplicationSystem` | 5 |

`ObjectReplicationBridge` / `ReplicationBridge` / `ReplicationSystem` are the
core runtime singletons of Iris's object-replication system. Their presence
in the live `GObjects` array proves Iris is not merely compiled in but is
actually instantiated and running — i.e. it is the active replication
backend, not dead code.

### E3 — Build/version corroboration
`dumper-7/Mappings/5.6.0-31351+++Tyr+release-Tyr.usmap` confirms the dump
target is the Tyr release build on UE 5.6.0 (changelist 31351), matching the
stated engine version in `README.md`.

## Strength assessment
Both "compiled-in" (E1) and "runtime-instantiated" (E2) are satisfied. This
is the strongest form of *static* evidence available without a live
debugger. It is treated as settled per `README.md` / `00-overview-and-setup.md`
Step 0.1 and must not be re-litigated in later phases.
